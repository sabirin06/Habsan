1. Overview
The ATA Somalia GDS API enables third-party platforms to search for flights, create bookings, process payments, and manage reservations programmatically.
Base URLs

9. Complete Booking Flow
Here is the step-by-step sequence to book a flight via the API:

┌──────────────────────────────────────────────────────────────────────────┐

│                     COMPLETE BOOKING FLOW                               │

├──────────────────────────────────────────────────────────────────────────┤

│                                                                          │

│  STEP 1: Authenticate                                                    │

│  ────────────────────                                                    │

│  POST /authenticator/loginUser                                           │

│  → Save accessToken for subsequent requests                              │

│                                                                          │

│  STEP 2: Search Airports (optional, for UI)                              │

│  ──────────────────────────────────────────                              │

│  POST /Airport/search  { "search": "Mogadishu" }                        │

│  → Get IATA codes (MGQ, NBO, etc.)                                       │

│                                                                          │

│  STEP 3: Get Payment Options                                             │

│  ────────────────────────────                                            │

│  GET /FlightBooking/paymentOptions/microSiteClientId/2                   │

│  → Get available payment methods                                         │

│                                                                          │

│  STEP 4: Search Flights                                                  │

│  ──────────────────────                                                  │

│  POST /FlightQuote/search                                                │

│  → Returns search ID + list of flights with prices                       │

│  → Note: Save response.id (searchId) and selected flight.id (flightId)  │

│                                                                          │

│  STEP 5: Select Flight                                                   │

│  ─────────────────────                                                   │

│  (Client-side) User picks a flight from the search results               │

│  → Store searchId + flightId                                             │

│  → No separate API call needed                                           │

│                                                                          │

│  STEP 6: Create Booking                                                  │

│  ──────────────────────                                                  │

│  POST /FlightBooking/addBooking                                          │

│  → Send flightSelection, passengers, paymentTypeId                       │

│  → If response.type == "DirectBooking" → Booking is confirmed ✅        │

│  → If response.type == "RedirectUrl" → Proceed to Step 7                │

│                                                                          │

│  STEP 7: Process Payment (if needed)                                     │

│  ────────────────────────────────────                                    │
